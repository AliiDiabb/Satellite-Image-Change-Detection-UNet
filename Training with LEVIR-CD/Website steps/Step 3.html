🤖STEP 3: Model Training

2D-CNN (U-Net) for Satellite Image Change Detection

What is Model Training?

Model training is the process where the AI learns to detect building changes by analyzing thousands of satellite image pairs. Think of it like teaching a student: you show them many examples of "before" and "after" images with the correct answers (ground truth masks), and over time, the student learns to identify changes on their own.

In this step, our 2D-Convolutional Neural Network (U-Net) will learn from 447 training images, validate its learning with 64 validation images, and prove its knowledge on 126 test images to achieve our target of 70% accuracy.

🧠 U-Net Architecture (2D-CNN)

Why U-Net for Change Detection?

U-Net is a special type of Convolutional Neural Network designed for pixel-level predictions. It's perfect for our task because it can identify exactly which pixels changed (new buildings) while preserving spatial information through skip connections.

📉 Encoder (Downsampling)

  • Block 1: 64 filters
  • Block 2: 128 filters
  • Block 3: 256 filters
  • Block 4: 512 filters
  • Purpose: Extract features

🔥 Bottleneck

  • Filters: 1024
  • Size: 16×16 (compressed)
  • Purpose: Deep understanding
  • Function: Learns abstract patterns

📈 Decoder (Upsampling)

  • Block 1: 512 filters + skip
  • Block 2: 256 filters + skip
  • Block 3: 128 filters + skip
  • Block 4: 64 filters + skip
  • Output: 256×256 mask

📊 Evaluation Metrics

How We Measure Performance

Our model is evaluated using multiple metrics to ensure comprehensive performance assessment

🎯
Accuracy
Percentage of correctly classified pixels (changed vs unchanged)
🔍
Precision
Of all predicted changes, how many were actually changes?
📡
Recall
Of all actual changes, how many did we detect?
⚖️
F1-Score
Harmonic mean of precision and recall (balanced measure)
🔲
IoU
Intersection over Union - overlap between prediction and ground truth
🎲
Dice Coefficient
Similar to IoU, measures similarity between prediction and truth

💡 Tips for Successful Training

Best Practices

🔌 Keep Power Connected

Training takes 6-10 hours. Keep your laptop plugged in and prevent sleep mode.

🌡️ Monitor GPU Temperature

Use nvidia-smi to check GPU usage. Ensure good ventilation.

💾 Close Other Programs

Close browsers, games, and heavy applications to maximize GPU memory.

📊 Check Progress Regularly

Monitor validation accuracy. If it plateaus, early stopping will activate.

🔄 Use Overnight Training

Start training before bed, check results in the morning.

💿 Backup Your Work

Save preprocessed data and training scripts to cloud storage.

⚠️ Troubleshooting Common Issues

Common Problems & Solutions

  • Out of Memory Error: Reduce BATCH_SIZE from 8 to 4 or even 2 in Config class
  • Training Very Slow: Check if GPU is being used with nvidia-smi command
  • Accuracy Stuck Below 70%: Train for more epochs (increase EPOCHS to 100)
  • Model Not Improving: Check if validation loss is decreasing. May need to adjust learning rate
  • File Not Found Error: Verify DATA_PATH points to correct preprocessed folder
  • GPU Not Detected: Ensure CUDA and cuDNN are properly installed for TensorFlow 2.10
