🔧 Step 2: Dataset Preprocessing Explained

Understanding How We Prepare Satellite Images for AI Training

What is Preprocessing?

Preprocessing is the process of transforming raw satellite images into a clean, uniform format that our AI model can understand and learn from efficiently. Think of it as preparing ingredients before cooking - you wash, cut, and organize everything so the actual cooking (training) goes smoothly!

1
📏
Resizing Images

We reduce the image size from very large to a manageable dimension that our AI can process efficiently.

What Changes:
📐 Original size: 1024×1024 pixels (very large!)
📐 After preprocessing: 256×256 pixels (smaller, manageable)
💡 Why This Matters: Large images require too much GPU memory and training time. Smaller images train 10x faster while keeping important details like building edges and structures intact.
2
🎨
Image Format Conversion

We convert images from OpenCV's BGR format to the standard RGB format that neural networks expect.

What Changes:
🔄 BGR (Blue-Green-Red) → RGB (Red-Green-Blue)
🌈 Ensures colors are displayed correctly
💡 Why This Matters: Neural networks are trained on RGB images (standard format). Without this conversion, the model would see all colors incorrectly - like looking at a photo through colored glasses!
3
⚫⚪
Mask Binarization

We convert grayscale labels into pure black-and-white masks - making it crystal clear where buildings changed.

What Changes:
⚫ 0 (Black) = No change detected
⚪ 255 (White) = Building change detected
🚫 All gray values removed (converted to 0 or 255)
💡 Why This Matters: The model needs clear yes/no answers, not shades of gray. Binary masks make it easy for the AI to learn: "Did a building change here? Yes or No."
4
📦
File Organization

We organize all images into three separate folders for proper AI training workflow.

Folder Structure:
📁 train/ - For teaching the model (447 images)
📁 val/ - For checking progress during training (64 images)
📁 test/ - For final evaluation (126 images)
💡 Why This Matters: Separating data prevents the model from "cheating" by seeing test data during training. It's like taking an exam - you can't study using the actual test questions!

🍳 Simple Cooking Analogy

Think of preprocessing like preparing ingredients before cooking a meal:

✂️

Resizing

Cutting vegetables to uniform size for even cooking

🧼

Format Conversion

Washing and cleaning ingredients to remove dirt

📊

Binarization

Sorting ingredients into "use" or "don't use" piles

🥘

Organization

Arranging ingredients in separate bowls for each cooking step

⚖️ Without vs With Preprocessing

❌ Without Preprocessing

  • Training is 10x slower (huge images)
  • Model might run out of memory
  • Inconsistent data formats cause errors
  • Poor model accuracy
  • Can't compare results properly

✅ With Preprocessing

  • Fast, efficient training
  • Consistent input format
  • Optimal memory usage
  • Better model performance
  • Easy to work with and debug

Preprocessing transforms raw, large, inconsistent satellite images into clean, uniform, AI-ready data that your model can learn from efficiently.
